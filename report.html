1. 算法概述

1.1 Cox-Ross-Rubinstein (CRR) 二叉树模型

二叉树模型是期权定价的经典数值方法。将期权到期时间 T 分成 N 个等长的时间步，每个节点处标的资产价格向上或向下移动：

参数公式说明
时间步长Δt = T / N
上升因子u = exp(σ√Δt)
下降因子d = 1 / u保证重组树
风险中性概率p = (exp(rΔt) - d) / (u - d)
折现因子disc = exp(-rΔt)每步折现

1.2 美式 vs 欧式期权的区别

欧式期权：只能在到期日行权。倒推时直接折现。

美式期权：可在任意时刻提前行权。倒推时需比较「持有价值（折现）」和「立即行权收益」，取较大值。

金融理论验证点：对于不付红利的标的资产，美式看涨期权不会提前行权，因此美式看涨 = 欧式看涨。而美式看跌 > 欧式看跌。

3. Oracle PL/SQL 与 GaussDB PL/pgSQL 差异要点

本节记录了在将 Oracle PL/SQL 存储过程移植到 GaussDB PL/pgSQL 过程中遇到的关键差异。

3.1 REVERSE FOR 循环语法

核心差异：GaussDB 支持 REVERSE 关键字，但两个边界参数的顺序与 Oracle 相反。
数据库语法说明
Oracle FOR i IN REVERSE 0..p_n-1 LOOP 小数在前，大数在后
GaussDB FOR i IN REVERSE p_n-1..0 LOOP 大数在前，小数在后

Oracle 的 REVERSE 0..5 等价于 GaussDB 的 REVERSE 5..0，都产生 5,4,3,2,1,0 序列。但两者的参数顺序恰好相反。

3.2 变量声明格式

Oracle PL/SQLGaussDB PL/pgSQL
v_dt NUMBER; v_u NUMBER; (分号结尾)v_dt DOUBLE PRECISION; (分号结尾)
同一行声明多个变量用分号隔开每个变量独立声明

3.3 数组类型与初始化

特性OracleGaussDB
关联数组类型TYPE arr IS TABLE OF NUMBER INDEX BY PLS_INTEGER;DOUBLE PRECISION[]
初始化自动扩展，无需预分配需循环预分配：FOR j IN 1..n+1 LOOP v_opt[j]:=0.0; END LOOP;
索引起点0-based (v_opt(0))1-based (v_opt[1])

3.4 类型严格性

GaussDB 对类型匹配更严格。例如 GREATEST(v_stk - p_k, 0) 会报错，必须写成 GREATEST(v_stk - p_k, 0.0)，否则抛出 "operator does not exist: double precision > integer"。

3.5 函数 vs 过程

OracleGaussDB
使用 PROCEDURE ... OUT NUMBER 返回值使用 FUNCTION ... RETURNS DOUBLE PRECISION
JDBC: CallableStatement + registerOutParameterJDBC: PreparedStatement + ResultSet

4. 数据库存储过程源码

4.1 Oracle PL/SQL 存储过程

CREATE OR REPLACE PROCEDURE binomial_option_price (
    p_s0      IN  NUMBER,
    p_k       IN  NUMBER,
    p_t       IN  NUMBER,
    p_r       IN  NUMBER,
    p_sigma   IN  NUMBER,
    p_n       IN  NUMBER,
    p_is_call IN  NUMBER,   -- 1=看涨, 0=看跌
    p_is_amer IN  NUMBER,   -- 1=美式, 0=欧式
    p_result  OUT NUMBER
) AS
    v_dt        NUMBER;
    v_u         NUMBER;
    v_d         NUMBER;
    v_p         NUMBER;
    v_disc      NUMBER;
    TYPE num_array IS TABLE OF NUMBER INDEX BY PLS_INTEGER;
    v_opt       num_array;
    i           PLS_INTEGER;
    j           PLS_INTEGER;
    v_intrinsic NUMBER;
    v_cont      NUMBER;
    v_stk       NUMBER;
BEGIN
    v_dt   := p_t / p_n;
    v_u    := EXP(p_sigma * SQRT(v_dt));
    v_d    := 1.0 / v_u;
    v_p    := (EXP(p_r * v_dt) - v_d) / (v_u - v_d);
    v_disc := EXP(-p_r * v_dt);

    -- 初始化到期日收益
    FOR j IN 0..p_n LOOP
        v_stk := p_s0 * POWER(v_u, p_n - j) * POWER(v_d, j);
        IF p_is_call = 1 THEN v_opt(j) := GREATEST(v_stk - p_k, 0);
        ELSE v_opt(j) := GREATEST(p_k - v_stk, 0); END IF;
    END LOOP;

    -- 倒推折现 (Oracle: REVERSE 小数..大数)
    FOR i IN REVERSE 0..p_n-1 LOOP
        FOR j IN 0..i LOOP
            v_cont := v_disc * (v_p * v_opt(j+1) + (1-v_p) * v_opt(j));
            IF p_is_amer = 1 THEN
                v_stk := p_s0 * POWER(v_u, i-j) * POWER(v_d, j);
                IF p_is_call = 1 THEN v_intrinsic := GREATEST(v_stk - p_k, 0);
                ELSE v_intrinsic := GREATEST(p_k - v_stk, 0); END IF;
                v_opt(j) := GREATEST(v_intrinsic, v_cont);
            ELSE
                v_opt(j) := v_cont;
            END IF;
        END LOOP;
    END LOOP;

    p_result := v_opt(0);
END;

4.2 GaussDB PL/pgSQL 存储函数

CREATE OR REPLACE FUNCTION binomial_amer_opt_v3(
    p_s0      DOUBLE PRECISION,
    p_k       DOUBLE PRECISION,
    p_t       DOUBLE PRECISION,
    p_r       DOUBLE PRECISION,
    p_sigma   DOUBLE PRECISION,
    p_n       INTEGER,
    p_is_call INTEGER,
    p_is_amer INTEGER
) RETURNS DOUBLE PRECISION AS $$
DECLARE
    v_dt        DOUBLE PRECISION;
    v_u         DOUBLE PRECISION;
    v_d         DOUBLE PRECISION;
    v_p         DOUBLE PRECISION;
    v_disc      DOUBLE PRECISION;
    v_opt       DOUBLE PRECISION[];
    i           INTEGER;
    j           INTEGER;
    v_cont      DOUBLE PRECISION;
    v_intrinsic DOUBLE PRECISION;
    v_stk       DOUBLE PRECISION;
BEGIN
    v_dt   := p_t / p_n;
    v_u    := EXP(p_sigma * SQRT(v_dt));
    v_d    := 1.0 / v_u;
    v_p    := (EXP(p_r * v_dt) - v_d) / (v_u - v_d);
    v_disc := EXP(-p_r * v_dt);

    -- GaussDB 需预分配数组
    FOR j IN 1..p_n+1 LOOP v_opt[j] := 0.0; END LOOP;

    -- 终端节点收益 (索引从 j+1 开始，因为 GaussDB 数组从 1 开始)
    FOR j IN 0..p_n LOOP
        v_stk := p_s0 * power(v_u, p_n - j) * power(v_d, j);
        IF p_is_call = 1 THEN v_opt[j+1] := GREATEST(v_stk - p_k, 0.0);
        ELSE v_opt[j+1] := GREATEST(p_k - v_stk, 0.0); END IF;
    END LOOP;

    -- 倒推折现 (GaussDB: REVERSE 大数..小数，与 Oracle 顺序相反!)
    FOR i IN REVERSE p_n-1..0 LOOP
        FOR j IN 0..i LOOP
            v_cont := v_disc * (v_p * v_opt[j+2] + (1.0 - v_p) * v_opt[j+1]);
            IF p_is_amer = 1 THEN
                v_stk := p_s0 * power(v_u, i - j) * power(v_d, j);
                IF p_is_call = 1 THEN v_intrinsic := GREATEST(v_stk - p_k, 0.0);
                ELSE v_intrinsic := GREATEST(p_k - v_stk, 0.0); END IF;
                v_opt[j+1] := GREATEST(v_intrinsic, v_cont);
            ELSE
                v_opt[j+1] := v_cont;
            END IF;
        END LOOP;
    END LOOP;

    RETURN v_opt[1];
END;
$$ LANGUAGE plpgsql;
